========================================
FaultPilot description
========================================

A RAG-powered assistant for industrial equipment troubleshooting.
Diagnoses alarm codes, fault symptoms, and PLC programming queries across Fanuc and Bosch documentation using hybrid search, cross-encoder reranking, and intelligent query routing.

========================================
Optional Features Implemented 
========================================

1. Implement streaming responses
   - Real-time chunked response updates in Gradio using `yield` from UI controllers.

2. The app is designed for a specific goal/domain that is not a tutor about AI
   - Industrial OT troubleshooting: Diagnoses alarm codes, fault symptoms, and PLC programming queries across Fanuc and Bosch documentation using hybrid search, cross-encoder reranking, and intelligent query routing.

3. You have shown evidence of collecting at least two data sources beyond those provided in our course
   - Two manufacturer manuals currently ingested and indexed from PDF:
        Fanuc AC Spindle Drive Alarm List
        Bosch CC 220/320 Error Messages

4. Your data collection and curation process leverages images and/or PDFs
   - Source data are industrial PDFs with mixed extraction challenges:
            + corrupted table encoding (Bosch)
            + alarm list parsing with code normalization (Fanuc)
        Custom parsers handle each case.

5. Your data collection and curation process leverages structured JSON outputs, which are used for advanced RAG functionalities in your app
   - Each chunk is enriched with structured metadata: 
            + alarm_code
            + manufacturer
            + equipment
            + category
            + language
            + page, source_doc. 
        These fields drive metadata filtering in the retrieval pipeline.

6. Use hybrid search in your RAG pipeline
   - BM25 (sparse, via rank_bm25) + dense embeddings (sentence-transformers/all-MiniLM-L6-v2) fused with Reciprocal Rank Fusion.
     Route profiles set per-intent retrieval depth.

7. Use a reranker in your RAG pipeline
   - Cross-encoder reranker (cross-encoder/ms-marco-MiniLM-L-6-v2) reranks fused candidates before final context build.

8. Use metadata filtering
   - Retrieval filters support manufacturer, equipment, and language.
   - Users can select manufacturer/equipment in the UI to narrow results.

9. Your RAG pipeline includes query routing
   - Dual-layer query router: local classifier + model-backed fallback.
   - Routes to alarm_lookup, troubleshooting, or programming.
